# Security Notes

Last updated: 2026-07-05

## Never commit

- `.env.local`, `.env`, production credentials
- `server/data/*.json` — local runtime users/sessions/households
- `.tmp-e2e-data/`, Playwright/Lighthouse reports
- Invite codes, session tokens, Dexcom OAuth tokens

Run before push: `npm run scan:secrets`

## Credentials requiring rotation if exposed

| Credential | Status | Action |
|------------|--------|--------|
| Neon `DATABASE_URL` password | **Assume compromised** if shared in chat/logs | Rotate in Neon Console → update `.env.local` + Vercel |
| `T1D_GOOGLE_CLIENT_SECRET` | In `.env.local` only (gitignored) | Rotate if ever committed |
| `T1D_CRON_SECRET` / `CRON_SECRET` | Regenerated by sync script if missing locally | Store in `.env.local` after first sync |
| Dexcom client secret | Placeholder locally | Rotate when real credentials added |

## Repository hygiene

- `.gitignore` excludes runtime artifacts
- Pre-push: `npm run scan:secrets` + `npm run test:unit`
- Health data in Postgres: encrypt Dexcom tokens at rest (`T1D_SECRETS_KEY`)
